# 403 Forbidden 错误排查指南

## 🚨 问题描述
访问 `http://resume.uied.cn/` 时出现 403 Forbidden 错误，服务器为 Tengine。

## 🔍 可能原因分析

### 1. Node.js 应用未正常启动 ⭐⭐⭐⭐⭐
**最常见原因**
- Node.js 进程可能已停止或崩溃
- 端口冲突导致应用无法启动
- 应用启动时出现错误

### 2. 文件权限问题 ⭐⭐⭐⭐
- 网站目录权限设置不正确
- server.js 文件没有执行权限
- .next 目录权限问题

### 3. 反向代理配置错误 ⭐⭐⭐
- 反向代理未正确配置
- 目标端口设置错误
- 代理规则配置问题

### 4. 防火墙或安全组限制 ⭐⭐
- 服务器防火墙阻止了端口访问
- 宝塔安全设置过于严格

## 🛠️ 排查步骤

### 第一步：检查 Node.js 应用状态
1. 登录宝塔面板
2. 进入 **软件商店** → **Node.js 版本管理器**
3. 查看项目列表，确认应用状态：
   - ✅ **运行中**：绿色状态
   - ❌ **已停止**：红色状态
   - ⚠️ **异常**：黄色状态

**如果应用已停止：**
```bash
# 手动启动应用
cd /www/wwwroot/resume.uied.cn
node server.js
```

### 第二步：检查应用日志
在宝塔面板中查看应用日志：
1. Node.js 项目管理 → 选择项目 → **日志**
2. 查看错误信息，常见错误：
   ```
   Error: listen EADDRINUSE :::3000  # 端口被占用
   Error: Cannot find module         # 依赖缺失
   Permission denied                 # 权限问题
   ```

### 第三步：验证文件权限
```bash
# 设置正确的文件权限
cd /www/wwwroot/resume.uied.cn
chmod -R 755 .
chmod +x server.js
chown -R www:www .
```

### 第四步：检查端口占用
```bash
# 检查 3000 端口是否被占用
netstat -tlnp | grep :3000
# 或者
lsof -i :3000
```

### 第五步：验证反向代理配置
在宝塔面板中检查网站设置：
1. **网站** → 选择域名 → **反向代理**
2. 确认配置：
   - 代理名称：任意
   - 目标URL：`http://127.0.0.1:3000`
   - 发送域名：`$host`
   - 内容替换：留空

### 第六步：测试本地访问
```bash
# 在服务器上测试本地访问
curl http://127.0.0.1:3000
# 应该返回 HTML 内容而不是错误
```

## 🔧 解决方案

### 方案一：重新部署应用
```bash
# 1. 停止现有应用
pm2 stop all  # 如果使用 PM2
# 或在宝塔面板中停止应用

# 2. 重新解压部署包
cd /www/wwwroot/resume.uied.cn
rm -rf * .*  # 清空目录（谨慎操作）
tar -xzf my-resume-builder-deploy.tar.gz

# 3. 设置权限
chmod -R 755 .
chown -R www:www .

# 4. 启动应用
node server.js
```

### 方案二：使用 PM2 管理进程
```bash
# 1. 安装 PM2（如果未安装）
npm install -g pm2

# 2. 创建 PM2 配置文件
cat > ecosystem.config.js << EOF
module.exports = {
  apps: [{
    name: 'resume-builder',
    script: 'server.js',
    cwd: '/www/wwwroot/resume.uied.cn',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    }
  }]
}
EOF

# 3. 启动应用
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

### 方案三：修改端口配置
如果 3000 端口被占用：
```bash
# 1. 修改环境变量
echo "PORT=3001" >> .env.local

# 2. 更新反向代理配置
# 在宝塔面板中将目标URL改为 http://127.0.0.1:3001

# 3. 重启应用
```

## 📋 检查清单

- [ ] Node.js 应用正在运行
- [ ] 应用日志无错误信息
- [ ] 文件权限设置正确 (755)
- [ ] 端口 3000 未被其他进程占用
- [ ] 反向代理配置正确
- [ ] 本地访问 `http://127.0.0.1:3000` 正常
- [ ] 防火墙允许相关端口访问
- [ ] 域名解析正确指向服务器

## 🆘 紧急恢复步骤

如果以上方法都无效，请按以下步骤操作：

1. **完全重新部署**：
   ```bash
   cd /www/wwwroot/resume.uied.cn
   rm -rf *
   # 重新上传并解压 my-resume-builder-deploy.tar.gz
   ```

2. **使用默认配置**：
   ```bash
   cp .env.example .env.local
   node server.js
   ```

3. **检查系统资源**：
   ```bash
   free -h  # 检查内存
   df -h    # 检查磁盘空间
   ```
