## 概述

性能监控工具提供了完整的页面性能测量功能，包括 Core Web Vitals 指标和自定义性能标记。

## 基础使用

### 1. 在应用入口初始化

```typescript
// src/app/layout.tsx
import { initPerformanceMonitor } from '@/utils/performanceMonitor';

export default function RootLayout({ children }) {
  useEffect(() => {
    // 初始化性能监控
    initPerformanceMonitor({
      enableConsoleLog: process.env.NODE_ENV === 'development',
      endpoint: '/api/performance', // 可选：上报端点
      sampleRate: 0.1, // 10% 采样率
    });
  }, []);

  return <html>{children}</html>;
}
```

### 2. 使用 React Hook

```typescript
// 在任何组件中使用
import { usePerformanceMonitor } from '@/hooks/usePerformanceMonitor';

function MyComponent() {
  const { metrics, isMonitoring } = usePerformanceMonitor({
    autoInit: true,
    enableConsoleLog: true,
  });

  return (
    <div>
      <p>FCP: {metrics.fcp?.toFixed(2)}ms</p>
      <p>LCP: {metrics.lcp?.toFixed(2)}ms</p>
      <p>监控状态: {isMonitoring ? '运行中' : '已停止'}</p>
    </div>
  );
}
```

### 3. 简化版 Hook（仅获取指标）

```typescript
import { usePerformanceMetrics } from '@/hooks/usePerformanceMonitor';

function PerformanceDisplay() {
  const metrics = usePerformanceMetrics();
  
  return (
    <div>
      {metrics.fcp && <p>FCP: {metrics.fcp.toFixed(2)}ms</p>}
      {metrics.lcp && <p>LCP: {metrics.lcp.toFixed(2)}ms</p>}
    </div>
  );
}
```

## 高级功能

### 自定义性能标记

```typescript
import { usePerformanceMonitor } from '@/hooks/usePerformanceMonitor';

function DataLoadingComponent() {
  const { mark, measure } = usePerformanceMonitor();

  const loadData = async () => {
    // 标记开始
    mark('data-load-start');
    
    await fetchData();
    
    // 标记结束
    mark('data-load-end');
    
    // 测量时间
    const duration = measure('data-load', 'data-load-start', 'data-load-end');
    console.log(`数据加载耗时: ${duration}ms`);
  };

  return <button onClick={loadData}>加载数据</button>;
}
```

### 测量组件渲染性能

```typescript
import { useRenderPerformance } from '@/hooks/usePerformanceMonitor';

function HeavyComponent() {
  const { startMeasure, endMeasure, duration } = useRenderPerformance('HeavyComponent');
  
  useEffect(() => {
    startMeasure();
    // 组件逻辑
    endMeasure();
  }, []);
  
  return <div>渲染时间: {duration}ms</div>;
}
```

### 手动上报指标

```typescript
function MyComponent() {
  const { reportMetrics } = usePerformanceMonitor();

  const handleExport = async () => {
    // 执行导出操作
    await exportResume();
    
    // 手动上报当前性能指标
    await reportMetrics();
  };

  return <button onClick={handleExport}>导出简历</button>;
}
```

## 性能指标说明

### Core Web Vitals

- **FCP (First Contentful Paint)**: 首次内容绘制时间
  - 良好: < 1.8s
  - 需要改进: 1.8s - 3.0s
  - 差: > 3.0s

- **LCP (Largest Contentful Paint)**: 最大内容绘制时间
  - 良好: < 2.5s
  - 需要改进: 2.5s - 4.0s
  - 差: > 4.0s

- **FID (First Input Delay)**: 首次输入延迟
  - 良好: < 100ms
  - 需要改进: 100ms - 300ms
  - 差: > 300ms

- **CLS (Cumulative Layout Shift)**: 累积布局偏移
  - 良好: < 0.1
  - 需要改进: 0.1 - 0.25
  - 差: > 0.25

### 其他指标

- **pageLoadTime**: 页面完整加载时间
- **domReady**: DOM 就绪时间
- **connectTime**: 服务器连接时间
- **resourceLoadTime**: 资源加载时间

## API 端点示例

如果需要上报性能数据到服务器，创建 API 端点：

```typescript
// src/app/api/performance/route.ts
import { NextRequest, NextResponse } from 'next/server';

export async function POST(request: NextRequest) {
  const data = await request.json();
  
  // 存储或处理性能数据
  console.log('Performance metrics:', data);
  
  // 可以存储到数据库或发送到分析服务
  // await saveToDatabase(data);
  
  return NextResponse.json({ success: true });
}
```

## 最佳实践

1. **生产环境使用采样**: 设置 `sampleRate` 避免过多请求
2. **使用 sendBeacon**: 工具会自动使用 sendBeacon API 避免阻塞页面卸载
3. **关键路径标记**: 为关键操作添加性能标记
4. **定期监控**: 在 CI/CD 中集成性能测试
5. **设置告警**: 当指标超过阈值时发送告警

## 注意事项

- 性能监控工具仅在浏览器环境中工作
- 某些旧浏览器可能不支持所有 API
- 开发环境的性能指标可能不准确，以生产环境为准
